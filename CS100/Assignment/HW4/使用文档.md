# CS100 Tank 游戏框架使用文档

本框架为终端版“坦克大战”游戏提供了完整的基础设施。为了完成作业，你需要了解并使用框架中提供的各种功能。以下是各个核心库（头文件）的功能、函数及使用方法的详细说明。

## 1. 基础工具库 (`Base.h`)

提供游戏开发中最基础的数据结构和工具函数。

- **二维向量 (`Vec`)**
  - `Vec` 结构体包含 `x` 和 `y` 两个整型成员，用于表示坐标或尺寸。
  - **运算函数**：`Add(a, b)`（加法）、`Sub(a, b)`（减法）、`Mul(a, b)`（乘法）、`Eq(a, b)`（判断相等）、`Neq(a, b)`（判断不等）。
- **随机数生成**
  - `Rand(int n)`：生成 `[0, n)` 范围内的随机整数。
  - `Rand01()`：生成 `[0, 1)` 范围内的随机浮点数。
  - `RandVec(Vec v)`：生成一个 `x` 在 `[0, v.x)`，`y` 在 `[0, v.y)` 范围内的随机向量。
- **颜色与文本**
  - 提供了一系列颜色宏，如 `TK_RED`（红）、`TK_GREEN`（绿）、`TK_NORMAL`（默认）等。
  - `TK_TEXT(...)`：用于格式化带颜色的文本输出。
- **时间控制**
  - `SleepMs(uint time)`：让当前线程休眠指定的毫秒数。
  - `Daze()`：极短时间的线程挂起，常用于帧率控制。

## 2. 游戏配置 (`Config.h`)

用于配置游戏初始参数。

- **`Config` 结构体**：包含 `fps`（帧率）、`mapSize`（地图尺寸）、`nEnemies`（敌人数量）、`nSolids`（坚固墙体数量）、`nWalls`（普通墙体数量）。
- **全局单例 `config`**：在调用 `GameLifecycle()` 之前设置此变量，`GameInit` 会根据它来生成场景。

## 3. 场景与物体 (`Scene.h`)

定义了游戏场景中的核心元素。

- **枚举类型**
  - `Dir`：表示坦克的朝向（如 `eDirOP` 向上，`eDirON` 向下，`eDirNO` 向左，`eDirPO` 向右等）。
  - `Flag`：表示地图上的静态物体（`eFlagNone` 空地，`eFlagSolid` 坚固墙体，`eFlagWall` 可破坏墙体）。
- **核心结构体**
  - `Tank`：坦克实体，包含位置 `pos`、方向 `dir`、颜色 `color` 和是否为玩家 `isPlayer`。
  - `Bullet`：子弹实体，包含位置 `pos`、方向 `dir`、颜色 `color` 和是否为玩家发射 `isPlayer`。
  - `Map`：地图实体，包含尺寸 `size` 和一维数组 `flags`（存储地图每个位置的静态物体）。
- **全局单例**
  - `regTank`：管理所有坦克的注册表。
  - `regBullet`：管理所有子弹的注册表。
  - `map`：全局地图实例。
- **辅助函数**
  - `Idx(Vec pos)`：将二维坐标 `pos` 转换为一维数组 `map.flags` 的索引。**使用方法**：`map.flags[Idx(pos)]`。
  - `RandPos()`：在地图范围内随机生成一个坐标。

## 4. 注册表系统 (`Registry.h`)

**黑魔法核心**：用于高效管理同类型结构体（如 `Tank` 和 `Bullet`）的集合。

- **创建与销毁实例**
  - `RegNew(reg)`：分配内存并自动将其加入注册表。例如：`Tank *tank = RegNew(regTank);`
  - `RegDelete(ptr)`：从注册表中移除并释放内存。例如：`RegDelete(tank);`
- **遍历注册表**
  - 使用迭代器 `RegIterator` 遍历所有实例。
  - **标准遍历范式**：
    ```c
    for (RegIterator it = RegBegin(regTank); it != RegEnd(regTank); it = RegNext(it)) {
        Tank *tank = RegEntry(regTank, it);
        // 在这里操作 tank，例如：tank->pos.y++;
    }
    ```
- **其他操作**
  - `RegSize(reg)`：获取当前注册表中的元素数量。

## 5. 渲染器 (`Renderer.h`)

负责将场景中的物体绘制到终端屏幕上。

- **核心绘制函数**
  - `RdrPutChar(Vec pos, char c, Color color)`：在指定位置 `pos` 绘制字符 `c`，颜色为 `color`。
  - 如果颜色传入 `TK_AUTO_COLOR`，渲染器会根据字符（如 `eFlagWall`）自动推导颜色。
- **渲染管线（在 `GameUpdate` 中使用）**
  1.  `RdrClear()`：清除上一帧的动态物体（坦克、子弹），将其覆盖为地图原本的静态背景。
  2.  **（你的逻辑代码）**：更新坦克和子弹的位置。
  3.  `RdrRender()`：将更新后的坦克和子弹绘制到当前帧的缓冲区中。
  4.  `RdrFlush()`：对比前后两帧，仅将发生变化的部分刷新到终端屏幕上。

## 6. 游戏生命周期 (`Game.h`)

控制游戏的整体运行流程。

- **全局单例 `game`**：包含 `keyHit` 成员，记录当前帧玩家按下的按键。
- **核心函数**
  - `GameInit()`：初始化终端、分配内存、生成地图和初始坦克。
  - `GameInput()`：读取玩家键盘输入，存入 `game.keyHit`。
  - `GameUpdate()`：**你需要重点修改的地方**。在这里编写坦克的移动、射击、碰撞检测等核心逻辑。
  - `GameTerminate()`：游戏结束时清理内存。
  - `GameLifecycle()`：游戏的主循环，按固定帧率依次调用上述函数。

## 7. 终端控制 (`Terminal.h`)

底层终端交互工具（通常不需要直接修改，框架已封装好）。

- `kbhit_t()`：检测是否有键盘输入。
- `getch_t()`：无回显地获取键盘输入的字符。
- `TermSetupGameEnvironment()`：配置终端为游戏模式（隐藏光标、禁用输入回显等）。

---

## 💡 综合使用建议（如何开始写作业）

1.  **处理输入与移动**：在 `Game.h` 的 `GameUpdate()` 中，检查 `game.keyHit`（如 'w', 'a', 's', 'd'），遍历 `regTank` 找到玩家坦克，修改其 `dir` 和 `pos`。
2.  **碰撞检测**：在移动坦克或子弹前，计算其下一个位置 `nextPos`，使用 `map.flags[Idx(nextPos)]` 检查该位置是否为 `eFlagSolid` 或 `eFlagWall`。
3.  **发射子弹**：当检测到射击按键（如 'k'）时，使用 `RegNew(regBullet)` 创建一颗新子弹，设置其初始位置为坦克前方，方向与坦克一致。
4.  **画面呈现**：在 `Renderer.h` 的 `RdrRender()` 中，修改坦克的绘制逻辑。目前只是画了一个 'O'，你需要使用两个 `for` 循环（3x3 范围）结合 `RdrPutChar` 画出具有方向感的坦克形状。
